Pulsar Clients

Pulsar exposes a client API with language bindings for Java and C++. Puslar 客戶端封裝並優化了 client-broker 的通信協議，並暴露出一套簡單直觀的 API，提供給應用程序使用。

底層實現上，目前官方版的 Pulsar 客戶端支持對用戶透明的連接重連、故障切換、未 ack 信息的緩衝、信息重傳。

自定義客戶端庫

如果你想創建自己的客戶端庫, 我們建議參考 Pulsar 的 binary protocol 文檔。

客戶端使用步驟

當應用程序要創建 producer/consumer 時, Pulsar 客戶端依照以下兩個步驟來進行設置:

  1. 客戶端將嘗試通過向服務器（Broker）發送 HTTP 查找請求，來確定主題（Topic）所在的服務器（Broker）。 客戶端通過查詢 Zookeeper 中（緩存）的元數據，來確定這條信息的 topic 在哪個 broker 上，如果該 topic 不在任何一個 broker 上，則把這個 topic 分配在負載最小的 broker 上。
  2. 當客戶端獲取了 broker 的地址之後，將會創建一個 TCP 連線（或複用連接池中的連線）並且進行身份認證。 客戶端和 broker 通過該連線交換基於自定義協議的二進制命令。 同時，客戶端會向 broker 發送一條命令用以在 broker 上創建producer/consumer，該命令將會在驗證授權策略後生效。

每當 TCP 連線中斷時, 客戶端將立即重新啟動連線握手階段, 並將繼續嘗試重新建立 producer/consumer, 直到操作成功為止。

Reader 介面接口

在 Pulsar 中, "標準" 的 consumer 介面接口 涉及監聽 topic, 處理傳入信息, 並在處理完這些信息後最終確認它們。 Whenever a consumer connects to a topic, it automatically begins reading from the earliest un-acked message onward because the topic's cursor is automatically managed by Pulsar.

使用 Pulsar 的 reader 介面接口, 應用程序可以手動管理 cursor。 當使用 reader 連接到一個 topic，當 reader 連接到 topic 的時候就需要指定 reader 從哪個位置開始讀信息。 當連接到某個 topic 時, reader 從以下位置開始讀信息:

  • 主題中的 **最早的 (earliest) ** 可用信息
  • 主題中的 **最新的 (latest) ** 可用信息
  • 除最早的和最新的之外的可用信息點位。 如果你使用該選項，你需要顯式的提供一個信息 ID。 你的應用程序需要自己管理信息 ID，例如從持久化的數據存儲或緩存中獲取。

使用 Pulsar 的 reader 介面接口, 應用程序可以手動管理 cursor。 在許多的使用情境，一個信息系統能否提供 topic 讀取倒帶 (rewind) 到指定的點位並且從那個位置開始重讀信息至為關鍵。 Reader 界面接口讓 Pulsar 客戶端在信息系統提供了一種底層抽象“手動管理點位”的實現。

只適用於 non-partitioned topics

目前 Pulsar 的 reader 接口介面無法在 partitioned topic 上使用。

Here's a Java example that begins reading from the earliest available message on a topic:

import org.apache.pulsar.client.api.Message;
import org.apache.pulsar.client.api.MessageId;
import org.apache.pulsar.client.api.Reader;

// Create a reader on a topic and for a specific message (and onward)
Reader<byte[]> reader = pulsarClient.newReader()
    .topic("reader-api-test")
    .startMessageId(MessageId.earliest)
    .create();

while (true) {
    Message message = reader.readNext();

    // Process the message
}

To create a reader that will read from the latest available message:

Reader<byte[]> reader = pulsarClient.newReader()
    .topic(topic)
    .startMessageId(MessageId.latest)
    .create();

To create a reader that will read from some message between earliest and latest:

byte[] msgIdBytes = // Some byte array
MessageId id = MessageId.fromByteArray(msgIdBytes);
Reader<byte[]> reader = pulsarClient.newReader()
    .topic(topic)
    .startMessageId(id)
    .create();
