Pulsar 2.0

Pulsar 2.0은 Pulsar의 새로운 주요한 릴리즈로, simplified topic names 과 Pulsar Functions 기능 및 일부 용어 변경 등을 포함하여 플랫폼의 많은 변경사항을 제공합니다.

주요 변경사항

일상적인 사용에 큰 영향을 줄 수 있는, 주의 할 몇 가지 주요 변경 사항이 있습니다.

Properties versus tenants

이전에, Pulsar는 properties라는 개념을 가지고 있었습니다. property은 기본적으로 tenent와 완전히 동일하므로, 버전 2.0에서 "property"용어가 제거되었습니다. 예를 들어 pulsar-admin properties</ code></a> 명령 행 인터페이스는 <a href="/docs/ko/2.1.1-incubating/pulsar-admin # pulsar-admin-tenants"> <code>pulsar-admin tenents 인터페이스로 대체되었습니다. 경우에 따라 properties 용어가 여전히 사용 되지만 이제는 더 이상 사용 되지 않는 것으로 간주되어 향후 릴리즈에서는 완전히 제거 될 것입니다.

Topic 이름

버전 2.0 이전의 모든 Pulsar topic 은 다음과 같은 형식을 갖습니다:

{persistent|non-persistent}://property/cluster/namespace/topic

Pulsar 2.0에는 두 가지 중요한 변경 사항이 있습니다:

  • 더 이상 cluster component가 없습니다.
  • Properties 이름이 tenants로 변경되었습니다.
  • 유연한 이름 지정 시스템을 사용하여 많은 topic 이름을 줄일 수 있습니다.

클러스터 구성 요소가 없습니다

클러스터 구성 요소가 topic 이름에서 제거되었습니다. 따라서, 모든 topic 이름은 이제 다음 형식을 갖습니다:

{persistent|non-persistent}://tenant/namespace/topic

기존 topic은 이전 이름 지정 방식을 사용 할 수 있으며 수정 없이 계속 이용 할 수 있습니다.

유연한 toipc 이름 지정

Pulsar 2.0의 모든 topic 이름은 위와 동일한 패턴을 갖지만 이제는 단순화한 단축 이름을 사용 할 수 있습니다. 유연한 이름 지정 시스템은 기본 topic 유형, tenant 및 namespace가 있습니다:

Topic 관점Default
topic typepersistent
tenantpublic
namespacedefault

아래 표는 기본값으로 topic 이름을 자동 변환하는 몇 가지 예를 보여줍니다:

입력 topic 이름변환 된 topic 이름
my-topicpersistent://public/default/my-topic
my-tenant/my-namespace/my-topicpersistent://my-tenant/my-namespace/my-topic

non-persistent topics의 경우 항상 full topic name을 계속 지정해야합니다. 현재 위의 persistent topic name에 대한 기본-변환 규칙은 적용되지 않습니다. 따라서, non-persistent://my-topic 과 같은 단축 이름을 사용 할 수 없으며 대신 non-persistent://public/default/my-topic을 사용 해야 합니다.
