Managing Connectors

本节描述如何在 Pulsar 集群中管理 Pulsar IO 连接器。你将学习如何：

  • 部署内置连接器
  • 借助 Pulsar Admin CLI （管理命令行工具）监控和更新运行中的连接器
  • 部署定制连接器
  • 更新连接器

使用内置连接器

Pulsar 捆绑了几个可以在数据库，消息系统等常用系统中移入，移出数据的 内置连接器。 这些内置连接器的配置与使用很简单。 你可以按照有关内置连接器的 说明 来进行操作。 配置完成后，Pulsar broker（或 function-worker）会自动发现所有的内置连接器，因此不需要额外的安装步骤。

配置连接器

配置 Pulsar IO 连接器是很简单的。 你只需要在 运行连接器 时提供 yaml 配置文件即可。 Yaml 配置文件会告诉 Pulsar 在哪里可以找到 source 和 sink，以及如何将这些 source 和 sink 与 Pulsar 主题连接起来。

下面是 Cassandra Sink 的 yaml 配置文件示例：

tenant: public
namespace: default
name: cassandra-test-sink
...
# cassandra specific config
configs:
    roots: "localhost:9042"
    keyspace: "pulsar_test_keyspace"
    columnFamily: "pulsar_test_table"
    keyname: "key"
    columnName: "col"

上面的 yaml 示例告诉 Pulsar 连接哪个 Cassandra 集群；Cassandra 中用于收集数据的 keyspace 和 columnFamily 是什么；以及如何将 Pulsar 消息映射到 Cassandra 表的 key 和 columns。

如果想要更详细的解释，请参阅 单个连接器 的文档。

运行连接器

Pulsar 连接器的管理可以使用 source 和 sink 命令。这些命令来自于 pulsar-admin CLI（命令行客户端）工具。

运行 source

你可以使用以下形式的命令提交一个在现有 Pulsar 集群中运行的 source：

$ ./bin/pulsar-admin source create --className  <classname> --jar <jar-location> --tenant <tenant> --namespace <namespace> --name <source-name> --destinationTopicName <output-topic>

这是一个示例命令：

bin/pulsar-admin source create --className org.apache.pulsar.io.twitter.TwitterFireHose --jar ~/application.jar --tenant test --namespace ns1 --name twitter-source --destinationTopicName twitter_data

除了在现有 Pulsar 集群上提交 source，你也可以在本地机器上将 source 作为进程运行：

bin/pulsar-admin source localrun --className  org.apache.pulsar.io.twitter.TwitterFireHose --jar ~/application.jar --tenant test --namespace ns1 --name twitter-source --destinationTopicName twitter_data

If you are submitting a built-in source, you don't need to specify --className and --jar. 可以简要指定 source 类型 --source-type。 提交内置 source 的命令可以是以下形式：

./bin/pulsar-admin source create \
    --tenant <tenant> \
    --namespace <namespace> \
    --name <source-name> \
    --destinationTopicName <input-topics> \
    --source-type <source-type>

下面是一个提交 Kafka source 的示例：

./bin/pulsar-admin source create \
    --tenant test-tenant \
    --namespace test-namespace \
    --name test-kafka-source \
    --destinationTopicName pulsar_sink_topic \
    --source-type kafka

运行 Sink

你可以使用以下形式的命令提交一个在现有 Pulsar 集群中运行的 sink：

./bin/pulsar-admin sink create --className  <classname> --jar <jar-location> --tenant test --namespace <namespace> --name <sink-name> --inputs <input-topics>

这是一个示例命令：

./bin/pulsar-admin sink create --className  org.apache.pulsar.io.cassandra --jar ~/application.jar --tenant test --namespace ns1 --name cassandra-sink --inputs test_topic

除了在现有 Pulsar 集群上提交 sink，你也可以在本地机器上将 sink 作为进程运行：

./bin/pulsar-admin sink localrun --className  org.apache.pulsar.io.cassandra --jar ~/application.jar --tenant test --namespace ns1 --name cassandra-sink --inputs test_topic

If you are submitting a built-in sink, you don't need to specify --className and --jar. 可以简要指定 sink 类型 --sink-type。 提交内置 sink 的命令可以是以下形式：

Note

当前内置连接器 sink-type 的参数是由 pulsar-io.yaml 文件中指定的 name 参数的设置决定的。

./bin/pulsar-admin sink create \
    --tenant <tenant> \
    --namespace <namespace> \
    --name <sink-name> \
    --inputs <input-topics> \
    --sink-type <sink-type>

下面是一个提交 Cassandra sink 的示例：

./bin/pulsar-admin sink create \
    --tenant test-tenant \
    --namespace test-namespace \
    --name test-cassandra-sink \
    --inputs pulsar_input_topic \
    --sink-type cassandra

监视连接器

Since Pulsar IO connectors are running as Pulsar Functions, so you can use functions commands available in the pulsar-admin CLI tool.

检查连接器元数据

bin/pulsar-admin functions get \
    --tenant <tenant> \
    --namespace <namespace> \
    --name <connector-name>

检查连接器运行状态

bin/pulsar-admin functions getstatus \
    --tenant <tenant> \
    --namespace <namespace> \
    --name <connector-name>
