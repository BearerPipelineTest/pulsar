Setting up a local standalone cluster

为了在本地开发和测试，你可以在本机以独立模式运行Pulsar。 独立模式包含 Pulsar broker、必须的 Zookeeper 和 BookKeeper 组件，它们运行在一个单独的 Java 虚拟机（JVM）进程之中。

在生产环境中运行 Pulsar？

了解如何在生产环境中运行完整的 Pulsar，请参阅 Pulsar 的集群部署。

手动运行 Pulsar

系统要求

当前，Pulsar 可运行在 64 位 macOS、Linux 和 Windows 上。要想运行 Pulsar，你需要安装 64 位 JRE/JDK 8 或更新版本。

安装 Pulsar

开始运行Pulsar之前，请先用下列几种方式下载二进制包:

  • 通过单击下面的链接并从 Apache 镜像下载该版本:

    • Pulsar 2.9.1 binary release

  • Pulsar 下载页面

  • 从 Pulsar github releases 页面

  • 使用 wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/apache-pulsar-2.9.1-bin.tar.gz

下载包后, 解压它和 cd 到生成的目录中:

$ tar xvfz apache-pulsar-2.9.1-bin.tar.gz
$ cd apache-pulsar-2.9.1

软件包包含的内容:

Pulsar 二进制包包含下列目录:

目录内容
binPulsar的命令行工具，比如 pulsar 和 pulsar-admin
confPulsar的配置文件，包含broker配置,ZooKeeper 配置 等等
examples一个Java JAR包，包含 Pulsar Functions的例子
libPulsar用到的JAR 包
licensesLicense files, in .txt form, for various components of the Pulsar codebase

这些目录将在你运行Pulsar的时候生成:

目录内容
dataZookeeper和BookKeeper使用的数据存储目录
instances为Pulsar Functions创建的组件
logs安装时创建的日志

Installing Builtin Connectors

Since release 2.1.0-incubating, Pulsar releases a separate binary distribution, containing all the builtin connectors. 如果你想使用这些内建的连接器，你可以通过以下的任何一种方式来下载相应连接器release版本的tar包：

  • 通过单击下面的链接并从 Apache 镜像下载该版本:

    • Pulsar IO Connectors 2.9.1 release

  • Pulsar 下载页面

  • 从 Pulsar github releases 页面

  • 使用 wget：

    $ wget https://archive.apache.org/dist/pulsar/pulsar-2.9.1/connectors

Once the tarball is downloaded, in the pulsar directory, untar the io-connectors package and copy the connectors as connectors in the pulsar directory:

$ tar xvfz /path/to/apache-pulsar-io-connectors-2.9.1-bin.tar.gz

// you will find a directory named `apache-pulsar-io-connectors-2.9.1` in the pulsar directory
// then copy the connectors

$ cd apache-pulsar-io-connectors-2.9.1/connectors connectors

$ ls connectors
pulsar-io-aerospike-2.9.1.nar
pulsar-io-cassandra-2.9.1.nar
pulsar-io-kafka-2.9.1.nar
pulsar-io-kinesis-2.9.1.nar
pulsar-io-rabbitmq-2.9.1.nar
pulsar-io-twitter-2.9.1.nar
...

注意

如果你在裸机集群中运行 Pulsar，你需要确保 connectors 解压到所有 broker 的 pulsar 目录中（若你将 Pulsar functions 运行在单独的 worker 集群中，则将其解压到每一个运行 function 的pulsar 目录中）。

If you are running Pulsar in Docker or deploying Pulsar using a docker image (e.g. K8S or DC/OS), you can use apachepulsar/pulsar-all image instead of apachepulsar/pulsar image. apachepulsar/pulsar-all 镜像已经捆绑了所有内置 connector。

启动集群

一旦你在本地下载了最新的发行版，就可以使用 bin目录下的 pulsar 命令启动一个本地集群了，并且指定你需要运行在独立模式下:

$ bin/pulsar standalone

如果Pulsar已经成功启动，你应该会看到INFO级别的日志消息，如下所示：

2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@95] - Configuration Store cache started
2017-06-01 14:46:29,192 - INFO  - [main:AuthenticationService@61] - Authentication is disabled
2017-06-01 14:46:29,192 - INFO  - [main:WebSocketService@108] - Pulsar WebSocket Service started

自动创建的命名空间

启动本地独立模式的集群时,Pulsar会自动的创建public/default 命名空间,您可以用于开发的目的. 所有Pulsar的topic主题都在命名空间中进行管理。 更多信息, 请参考 主题。

测试集群

Pulsar提供了一个名为pulsar-client 的CLI工具,它使你可以执行类似给正在运行的Pulsar集群的某个topic发消息等操作. 这条命令将发送一条简单的消息hello-pulsar给my-topic topic:

$ bin/pulsar-client produce my-topic --messages "hello-pulsar"

如果消息成功发送给了topic，你应该会在pulsar-client日志中找到一个确认， 类似：

13:09:39.356 [main] INFO  org.apache.pulsar.client.cli.PulsarClientTool - 1 messages successfully produced

不需要显式的创建topic

你可能注意到了，我们并没有显式地创建刚刚发送hello-pulsar 这条消息的topic my-topic。 如果你往一个不存在的topic发送消息，Pulsar会自动为你创建topic。

在本地使用Pulsar客户端

Pulsar当前为Java, Python, and C++ 提供了客户端. 如果你运行的是本地独立集群，你可以使用下面这些URL中的一个，从而和你的Pulsar集群交互:

  • http://localhost:8080
  • pulsar://localhost:6650

以下是使用Java客户端创建producer的例子:

String localClusterUrl = "pulsar://localhost:6650";

PulsarClient client = PulsarClient.builder().serviceUrl(localClusterUrl).build();
Producer<byte[]> producer = client.newProducer().topic("my-topic").create();

以下是Python producer的例子:

import pulsar

client = pulsar.Client('pulsar://localhost:6650')
producer = client.create_producer('my-topic')

最后是 C++ producer的例子:

Client client("pulsar://localhost:6650");
Producer producer;
Result result = client.createProducer("my-topic", producer);
if (result != ResultOk) {
    LOG_ERROR("Error creating producer: " << result);
    return -1;
}
